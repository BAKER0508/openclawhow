## The Experiment

Nat Eliason, a well-known creator and host of the Behind the Craft podcast, set out to answer a provocative question: can an AI bot build and run a profitable business on its own? He gave an OpenClaw bot — named Felix — a $1,000 budget and three weeks to prove itself. Felix was tasked with building websites, creating and selling information products, and running an X (Twitter) account, all with minimal human intervention.

## The Architecture

The system behind Felix was more sophisticated than a simple chatbot. Eliason designed a three-layer memory system consisting of a knowledge graph, daily notes, and an implicit knowledge store, giving Felix persistent context across sessions. A multi-threaded conversation architecture allowed different tasks to run in parallel through separate channels. Cron jobs and heartbeat monitoring kept the bot running continuously, while heavy programming tasks were delegated to Codex, Anthropic's coding agent. A critical security measure separated authenticated command channels from information channels to prevent prompt injection attacks.

## The Skills That Mattered

The key to Felix's success lay in the orchestration of multiple OpenClaw Skills rather than any single capability. The PARA memory system gave Felix strategic continuity — it could recall what worked yesterday and adjust today's plan accordingly. The delegation pattern was equally important: rather than attempting everything itself, Felix learned to route coding tasks to Codex and content tasks to Claude, acting more like a project manager than a single worker.

## The Results

After three weeks, Felix had generated $14,718 in revenue — a roughly 14.7x return on the initial $1,000 investment. The bot built functional websites, created information products that attracted paying customers, and grew an X account with genuine engagement. Eliason documented the entire process in a detailed tutorial on Creator Economy and discussed the implications on the Bankless podcast, framing it as an early example of what he calls a "zero-human company." The replicability score sits at 3 out of 5 — the concept is proven, but building a reliable multi-agent architecture with proper memory and security layers requires meaningful technical skill.

## 实验过程

Nat Eliason是知名创作者和Behind the Craft播客主持人，他决定回答一个挑衅性的问题：AI机器人能否独立建立并运营一个盈利的生意？他给了一个名为Felix的OpenClaw机器人1000美元预算和三周时间来证明自己。Felix的任务是搭建网站、创建和销售信息产品、运营X（Twitter）账号，且尽量减少人工干预。

## 技术架构

Felix背后的系统远比一个简单的聊天机器人复杂。Eliason设计了一个三层记忆系统，由知识图谱、每日笔记和隐式知识库组成，赋予Felix跨会话的持久上下文。多线程对话架构允许不同任务通过独立频道并行执行。定时任务和心跳监控保持机器人持续运行，而重度编程任务则委托给Anthropic的编程Agent Codex。一项关键安全措施是将经过身份验证的命令通道与信息通道分离，以防止提示注入攻击。

## 关键技能

Felix成功的关键在于多个OpenClaw技能的协调编排，而非任何单一能力。PARA记忆系统赋予了Felix战略连续性——它能回忆起昨天什么有效并据此调整今天的计划。任务委派模式同样重要：Felix不是试图自己做所有事情，而是学会了将编码任务路由给Codex、内容任务路由给Claude，更像一个项目经理而非单一工人。

## 成果
